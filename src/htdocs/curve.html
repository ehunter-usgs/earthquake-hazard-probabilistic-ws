Curve Endpoint Documentation

Probabilistic seismic hazard curve for one location.

Request Parameters

The curve end point requires certain parameters and accepts additional optional parameters as well.

Required

latitude
Latitude for location at which to compute a hazard curve. Decimal degrees.
longitude
Longitude for location at which to compute a hazard curve. Decimal degrees.
modelEdition
Hazard model edition to use when computing the hazard curve.
vs30
Soil conditions to use when computing the hazard curve

Optional

modelRegion
Default: Best available
Hazard model region to use when computing the hazard curve.
spectralPeriod
Default: All available
Spectral period of the hazard curve to compute.

Response

Responses follow the generalized response format. The documentation below describes the properties that will be found on the response.data attribute. For the curve endpoint, the response.data will be an array of Curve objects. Each curve object will have the following properties.

Curve Properties

metadata
Metadata describing the curve parameters.
latitude
Latitude of location for this hazard curve.
longitude
Longitude of location for this hazard curve.
modelEdition
Hazard model edition for this curve.
modelRegion
Hazard model region for this hazard curve.
spectralPeriod
Spectral period for this curve.
vs30
Soil conditions for this hazard curve.
data
Data defining a discrete hazard curve for the specified metadata properties. The data is formatted as an array of points. Each point is itself an array containing an [x, y] pair.

Example

Example Request URL

…curve.json?latitude=34.001&longitude=-118.001&modelEdition=E2014R1&vs30=760
