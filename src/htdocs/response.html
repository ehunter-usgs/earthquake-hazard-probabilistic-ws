Response Format Documentation

All web service responses come in a JSON format. At a high level, all the endpoints use a similar response format. In addition to the response body, all endpoints will serve HTTP status codes suitable to the response.

Properties

data
The requested data formatted as a JSON string. The specifics of this format vary depending on the individual endpoint being requested. If an error occurs, this value is populated with error information instead.
metadata
Metadata about the request/response formatted as a JSON string.
date
Request time as ISO8601 formatted string
status
One of “success” or “error” indicating the status of the response.
url
The requested URL including any query string parameters

Example Successful Reponse

{
  "data": {JSON Data},
  "metadata": {
    "date": "1970-01-01T00:00:00.000Z",
    "status": "success",
    "url": "{HOSTNAME}/{MOUNT_PATH}/curve.json?{parameters}"
  }
}

Example Error Response

{
  "data": "Internal Server Error",
  "metadata": {
    "date": "1970-01-01T00:00:00.000Z",
    "status": "error"
    "url": "{HOSTNAME}/{MOUNT_PATH}/curve.json?{parameters}"
  }
}
